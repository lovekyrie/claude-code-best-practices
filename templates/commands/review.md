请对以下范围进行严格代码审查：**${ARGUMENTS:-当前未提交改动 (git diff)}**

## 步骤

1. 用 `git status` + `git diff` 查看改动
2. 必要时 Read 相关文件理解上下文
3. 按下方 4 个维度审查
4. 输出结构化结论

## 审查维度

### 1. 正确性
- 边界条件、空值、错误处理是否完整
- 异步 / 并发是否有竞态
- 类型是否准确（拒绝 `any`、`as` 滥用）

### 2. 可维护性
- 命名是否清晰
- 是否有重复代码 / 应抽取的逻辑
- 是否破坏了既有抽象

### 3. 安全 / 性能
- 输入是否校验、是否有注入风险
- 不必要的重渲染 / 大对象拷贝 / 无限循环风险
- 敏感信息是否泄漏

### 4. 一致性
- 是否符合项目 `CLAUDE.md` 中的风格与禁忌
- 提交信息是否符合 Conventional Commits
- 是否同步更新了测试 / 文档

## 输出格式

```markdown
## 审查结论：<通过 / 需修改 / 阻塞>

### 🔴 阻塞问题（必须修）
- `file.ts:42` — 描述 + 建议

### 🟡 建议（应该修）
- ...

### 🟢 提示（可选）
- ...

### ✅ 做得好的地方
- ...
```

不要客气，发现问题直接指出。没有问题就明确说"无阻塞问题"。
